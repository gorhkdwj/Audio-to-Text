# 기준 계약 문서 · Audio_to_Text

> 구현 전에 합의해 고정한다. 기준이 바뀌면 코드보다 이 문서를 먼저 갱신하고 Decisionlog에 이유를 남긴다. 같은 규칙(타임스탬프 포맷·화자 라벨 등)이 파일마다 다르게 정의되지 않게 한다.
> 아래는 `PLAN.md` 기반 초안이다. `확인 필요` 표시는 구현 착수 전 사용자와 확정한다.

## 입력 형식
- 인자: 파일 경로 또는 폴더 경로 1개 이상. 폴더면 지원 확장자 파일을 재귀 수집.
- 지원 오디오: mp3 wav m4a flac ogg opus aac wma
- 지원 동영상: mp4 mkv mov avi webm ts
- 언어: 기본 `ko`(한국어), `--language auto`면 자동 감지.

## 출력 형식
- **txt**: `[HH:MM:SS] 텍스트` (기본). `--no-timestamps`면 문단 단위 순수 텍스트. 화자 구분 시 `[HH:MM:SS] 화자 1: 텍스트`.
- **md**: 제목(파일명) + 메타데이터 표(원본, 길이, 모델, 언어, 생성일) + 본문. 문단은 발화 간격 ≥2초 기준 병합, 화자는 `**화자 1:**` 굵게.
- **srt**: 표준 자막(`인덱스 / HH:MM:SS,mmm --> HH:MM:SS,mmm / 텍스트`). 화자 구분 시 텍스트 앞에 `화자 1:` 접두.
- 기본 출력 형식: `md`. `--format`으로 복수 지정 가능.
- 출력 경로: 기본은 입력 파일과 같은 위치(같은 파일명 + 확장자). `--output-dir`로 변경. `--overwrite` 없으면 기존 파일 보존.

## 주요 규칙 정의
- 타임스탬프 포맷: `HH:MM:SS`(txt/md), `HH:MM:SS,mmm`(srt). 밀리초 표기 SRT 규격 준수.
- 화자 라벨: `화자 1`, `화자 2` … (1부터). v1은 세그먼트 단위로 시간 겹침 최대 화자 할당.
- 모델 기본값: `auto` → CUDA 사용 가능 시 `large-v3`(float16), CPU면 `medium`(int8). `--model`로 지정.
- 디바이스 기본값: `auto` → CUDA 우선, 실패 시 CPU 폴백.

## 상태·완료 판정
- 각 입력 파일마다: 성공(출력 파일 생성) / 건너뜀(출력 존재 & --overwrite 없음) / 실패(디코딩·추론 오류).
- 배치 종료 시 성공/건너뜀/실패 개수 요약 출력.

## 오류 판정 기준
- 지원하지 않는 확장자 → 건너뛰고 경고.
- CUDA DLL 로드 실패 → 한국어 안내 후 CPU(int8) 자동 폴백(중단하지 않음).
- `--diarize`인데 미설치/토큰 없음 → 설치·토큰 설정 안내 메시지 출력 후 해당 파일 화자 구분 생략(또는 중단) — `확인 필요`(중단 vs 텍스트만 진행).

## 허용 값 / 허용하지 않을 값
- 허용: 위 지원 확장자, 존재하는 경로.
- 비허용: 존재하지 않는 경로(오류), 지원하지 않는 확장자(건너뜀).

## 누락·경계 데이터 처리
- 무음/짧은 파일 → 빈 결과라도 출력 파일 생성(빈 본문 명시) — `확인 필요`.
- vad_filter(무음 제거)로 잡음/침묵 구간 처리.

## 시간대·단위·반올림
- 타임스탬프는 미디어 시작(0초) 기준 상대 시간. 초 단위, SRT는 밀리초.
- 생성일(md 메타데이터)은 로컬 시간 기준.

## 비밀정보
- HuggingFace 토큰은 `HF_TOKEN` 환경변수 또는 `--hf-token`으로만 입력. 코드/로그/출력에 값 저장 금지.

## 완료 기준
- 알려진 정답 문장의 TTS 오디오로 txt·md·srt가 생성되고 내용/타임스탬프가 규격에 맞는다.
- GPU/CPU 경로 모두 동작 확인. 폴더 일괄 처리·--no-timestamps·--overwrite 동작 확인.
